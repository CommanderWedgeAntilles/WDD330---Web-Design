Errors, Exceptions, and Warnings

  • System error - there is a problem with the system or external devices
  • Programmer error - program contains incorrect syntax or faulty logic
  • User error - user entered the data wrong

Exceptions

An error that produces a return value that can be used to deal with the error. A non-existant method could result in a reference error -> and exception which will let the programmer know what is wrong.

Stack Traces

A sequence of functions or method calls that lead to the point where the error took place, produced by an exception. Stack trace works backward from the point where the error occurs to identify the origional function or method that started the sequence. Warnings

warnings are triggered when there is an issue which is not enough to crash the program. The program continues to run but the issue persists.

when a runtime error occurs in the browser, the html continues but the JavaScript code ceases to function. If a warning occurs, the JavaScript will continue to run.

Importance of Testing and Debugging

Because JavaScript fails quietly it is easy to miss mistakes. Make sure that the code you write fails loudly.

Strict Mode

'use strict'; is a a mode which forces the programmer to deal with mistakes early and often. It is the recommended means to write better quality JavaScript.

Linting Tools

  • Js Lint, JS Hint, Es Lint can be used to test the quality of JavaScript code, beyond using strict mode.
  • They are designed to highlight sloppy programming practices or syntax errors.
  • They can be unforgiving and use some opinionated coding conventions, such as not using the ++ and -- increment operators.
  • You can use online linting tools and test-editor plugins. you can also use npm to install linting software
